# 訓練和測試資料

## 資料夾結構

```
data/
├── train/              # 訓練資料（80%）
│   ├── 白尾八哥/       # 白尾八哥的圖片
│   ├── 家八哥/         # 家八哥的圖片
│   └── 林八哥/         # 林八哥的圖片
└── test/               # 測試資料（20%）
    ├── 白尾八哥/
    ├── 家八哥/
    └── 林八哥/
```

## 資料準備

### 方法 1: 執行準備腳本

```bash
cd src
python prepare_data.py
```

此腳本會：
1. 自動建立所需的資料夾結構
2. 顯示資料蒐集的說明
3. 檢查資料是否準備完成

### 方法 2: 手動建立

1. 在 `train/` 下建立三個類別資料夾
2. 在 `test/` 下建立三個類別資料夾
3. 將對應的鳥類照片放入資料夾中

## 資料需求

### 數量建議

- **訓練集（每類別）**: 50-100 張（最少 20 張）
- **測試集（每類別）**: 10-20 張（最少 5 張）

### 圖片要求

✅ **格式**: JPG, JPEG, PNG  
✅ **大小**: 最少 224x224 像素（會自動調整大小）  
✅ **品質**: 清晰、光線充足  
✅ **內容**: 鳥類主體明確可見  

### 資料多樣性

為了提升模型的泛化能力，圖片應包含：

- 不同角度（正面、側面、背面）
- 不同姿勢（站立、飛行、覓食）
- 不同環境（樹上、地面、空中）
- 不同光線（白天、陰天）
- 不同距離（近景、遠景）

## 資料來源建議

### 1. iNaturalist
- 網址: https://www.inaturalist.org/
- 特色: 開放授權的生物觀察照片
- 搜尋關鍵字:
  - "Acridotheres javanicus" (白尾八哥)
  - "Acridotheres tristis" (家八哥)
  - "Acridotheres grandis" (林八哥)

### 2. Flickr Creative Commons
- 網址: https://www.flickr.com/creativecommons/
- 特色: 可商用的開放授權照片
- 篩選: 選擇 CC BY 或 CC BY-SA 授權

### 3. Google 圖片搜尋
- 使用關鍵字: "白尾八哥"、"家八哥"、"林八哥"
- 工具 → 使用權限 → "創用 CC 授權"
- ⚠️ 仍需確認個別照片的授權

### 4. 自行拍攝（最推薦）
- 優點: 完全擁有版權、了解拍攝環境
- 地點: 公園、河濱、校園等
- 設備: 手機相機即可

## 資料擴增

訓練時會自動進行資料擴增（Data Augmentation）：

- ✅ 隨機旋轉（20度）
- ✅ 水平/垂直平移（20%）
- ✅ 剪切變換
- ✅ 隨機縮放（20%）
- ✅ 水平翻轉

這些技術可以從有限的資料中產生更多訓練樣本。

## 版權注意事項

⚠️ **重要**: 使用他人照片時請注意：

1. 確認照片有適當的授權（如創用CC）
2. 標註原作者和來源
3. 遵守授權條款
4. 避免使用有明顯浮水印的圖片
5. 本專案僅供教學使用

## 資料檢查清單

在開始訓練前，請確認：

- [ ] 每個類別至少有 20 張訓練圖片
- [ ] 圖片格式正確（JPG/PNG）
- [ ] 圖片清晰可見
- [ ] 檔案名稱無特殊字元
- [ ] 資料夾結構正確
- [ ] 已確認圖片授權

## 疑難排解

### 問題：資料夾建立失敗

```bash
# 手動建立資料夾
mkdir -p data/train/白尾八哥
mkdir -p data/train/家八哥
mkdir -p data/train/林八哥
mkdir -p data/test/白尾八哥
mkdir -p data/test/家八哥
mkdir -p data/test/林八哥
```

### 問題：中文資料夾名稱問題

- 確保系統支援 UTF-8 編碼
- 或改用英文資料夾名稱:
  - `javan_myna` (白尾八哥)
  - `common_myna` (家八哥)
  - `great_myna` (林八哥)

### 問題：圖片無法載入

- 檢查圖片是否損壞
- 使用圖片檢視器確認可以開啟
- 確認副檔名正確（.jpg 而非 .JPG）

## 進階技巧

### 1. 平衡資料集

確保每個類別的圖片數量相近，避免模型偏向某一類。

### 2. 清理異常值

移除：
- 模糊不清的圖片
- 鳥類主體太小的圖片
- 包含多種鳥類的圖片
- 圖片損壞或格式錯誤

### 3. 資料標註

如果圖片來源複雜，建議建立 CSV 記錄：
- 檔案名稱
- 類別
- 來源網址
- 授權資訊
